TripGuide v5.27 PUSH KEY READY

Deploy these files to the same GitHub Pages folder:
- index.html
- firebase-messaging-sw.js

This build inserts the Firebase Web Push public VAPID key into index.html.

After deployment:
1. Open the GitHub Pages TripGuide URL.
2. Sign in with an approved/admin account.
3. Click Enable Notifications.
4. Allow notifications.
5. Confirm a new entry appears in Firebase Realtime Database at:
   tripGuide/pushTokens/{uid}/{tokenId}

Do not put private Firebase service account credentials in this public web app. Actual push sending must be handled by a trusted backend such as Firebase Cloud Functions.

For iPhone/iPad testing, add the web app to the Home Screen first, then open it from the Home Screen and enable notifications there.

Firebase Cloud Functions sender setup (new):
- Files added: firebase.json, functions/package.json, functions/index.js
- Deploy Functions backend:
  1. npm --prefix functions install
  2. firebase login
  3. firebase use <your-project-id>
  4. firebase deploy --only functions:sendTripGuideNotification
- Required Firebase products/APIs:
  - Firebase Authentication
  - Realtime Database
  - Cloud Functions for Firebase
  - Firebase Cloud Messaging (FCM)

GitHub Actions manual sender setup (Blaze not required):
- Files added:
  - .github/workflows/send-tripguide-notification.yml
  - .github/scripts/send-tripguide-notification.js
- This manual sender does not require deploying Cloud Functions.
- Trigger method: GitHub Actions workflow_dispatch (Run workflow button)
- Notification behavior:
  - Reads tokens from tripGuide/pushTokens
  - Only sends to records where enabled=true and lastSeen (or createdAt) is within 90 days
  - Supports targets: allActive, adminsOnly, approvedViewers, singleUser
  - Automatically disables invalid or unregistered tokens
  - Prints attempted/success/failure/cleaned counts in the Actions logs

Required secure secret setup:
1. Firebase Console -> Project Settings -> Service Accounts
2. Generate new private key
3. GitHub repo -> Settings -> Secrets and variables -> Actions
4. Add secret FIREBASE_SERVICE_ACCOUNT_JSON
5. Paste entire JSON as the secret value
6. Run workflow manually from GitHub Actions tab

Security rules:
- Never commit Firebase service account JSON to this repo.
- Never expose private credentials in browser code or public files.
- Never expose server keys in index.html.
- Use only GitHub Actions secrets for FIREBASE_SERVICE_ACCOUNT_JSON.

Optional troubleshooting:
- If your Realtime Database is not reachable on the default URL pattern, add FIREBASE_DATABASE_URL as an additional GitHub Actions secret.
