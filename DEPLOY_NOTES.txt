TripGuide deployment notes (GitHub Actions-first push notifications)

Deploy these public files to GitHub Pages:
- index.html
- firebase-messaging-sw.js

Required model:
- Browser app only stores public Firebase config + public web VAPID key.
- Required push sending runs from GitHub Actions with firebase-admin.
- Cloud Functions sender in the UI is optional/legacy only.

Required GitHub Actions secrets:
1) FIREBASE_SERVICE_ACCOUNT_JSON (required)
   - Firebase Console -> Project Settings -> Service Accounts -> Generate private key
   - Store entire JSON in this Actions secret.
2) FIREBASE_DATABASE_URL (optional)
   - Only needed if your RTDB URL differs from the default project_id pattern.

Manual sender workflow:
- Workflow: .github/workflows/send-tripguide-notification.yml
- Script: .github/scripts/send-tripguide-notification.js
- workflow_dispatch inputs:
  - title (required)
  - body (required)
  - url (optional, default /TripGuide/)
  - targetType (required): allActive | adminsOnly | approvedViewers | singleUser
  - uid (optional; required when targetType=singleUser)

Manual sender behavior:
- Reads tripGuide/pushTokens
- Sends only enabled === true tokens
- Skips stale tokens older than 90 days by lastSeen/createdAt
- Targeting:
  - allActive: all active tokens
  - adminsOnly: built-in Mike/Lauren admin emails + tripGuide/admins/{uid}.admin === true (+ approvedUsers admin fallback)
  - approvedViewers: tripGuide/approvedUsers/{uid}.approved === true
  - singleUser: only provided uid
- Uses firebase-admin messaging from Actions only
- Sends data payload: { title, body, url }
- Disables invalid/unregistered tokens in RTDB (enabled=false + cleanup metadata)
- Logs attempted/success/failure/cleaned counts

Auto notifications:
- Workflow: .github/workflows/tripguide-auto-notifications.yml
- Script: .github/scripts/tripguide-auto-notifications.js
- Runs in GitHub Actions only with firebase-admin
- Uses notificationState de-dupe logic and only sends on state changes
- Covers:
  - day started
  - day complete
  - trip complete
  - live stale / live restored
  - access request
  - blocked user
  - trip reset
  - new message
- No Firebase Blaze plan requirement for this GitHub Actions path.

Realtime Database rules and indexes:
- Rules file: database.rules.json
- firebase.json includes:
  - "database": { "rules": "database.rules.json" }
- push token permissions and validation are defined for:
  - tripGuide/pushTokens/{uid}/{tokenId}
- messages index is defined:
  - tripGuide/messages -> .indexOn ["createdAt"]

Enable notifications in browser:
1) Sign in first (required).
2) Click Enable Notifications.
3) Allow browser notifications.
4) Confirm token record appears at:
   tripGuide/pushTokens/{uid}/{tokenId}

Platform notes:
- Android/Desktop: can enable directly in the browser.
- iPhone/iPad: must Add to Home Screen, open from Home Screen, then enable notifications.

How to test manual sender targets:
1) In GitHub -> Actions -> Send TripGuide Notification -> Run workflow.
2) Test allActive:
   - targetType=allActive
   - leave uid empty
   - verify enabled active devices receive message.
3) Test adminsOnly:
   - targetType=adminsOnly
   - verify only admin devices receive message.
4) Test approvedViewers:
   - targetType=approvedViewers
   - verify approved viewer devices receive message.
5) Test singleUser:
   - targetType=singleUser
   - set uid to a valid Firebase Auth uid
   - verify only that user’s device receives message.

How to verify invalid-token cleanup:
1) Use a known stale/invalid token (or a token from an uninstalled profile).
2) Run manual sender.
3) Check workflow logs for cleaned count > 0.
4) Confirm token record in tripGuide/pushTokens now has enabled=false and cleanup fields.

Security reminders:
- Never commit service account JSON.
- Never expose server keys or private credentials in index.html or service worker.
- Keep server-side send logic in GitHub Actions scripts only.
